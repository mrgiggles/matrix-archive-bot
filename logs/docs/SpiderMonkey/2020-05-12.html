errors in the JS spec don't automatically "unwind the stack" of standard algorithms
17:29

17:30
<jorendorff>
☝︎this is what's actually going on in the spec. Algorithms can return Completion Records. If the [[Type]] of a Completion Record is error then we know something went wrong.
17:31
<jorendorff>
And JS exceptions are implemented this way -- eeeevvvverything in the spec actually checks the return value after every little step
17:31
<jorendorff>

17:32
<jorendorff>
