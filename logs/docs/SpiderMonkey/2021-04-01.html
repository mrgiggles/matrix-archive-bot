<chaircrusher>
OK this is a good one: In SpiderMonkey 1.8.5 we do a thing where we keep a list of pointers to strings made with JS_NewExternalString. These are then assigned to JS object properties or array elements. Then later in our JSNative function, we get a JSString * from the values passed in, and look them up in our list of strings, to get the 'out of band' data we want about that string. In JS60 (and presumable earlier & later) this doesn't work. We assign these to JS String values & JSObject porperties, but later when we try to look up the strings, they're no longer the string we made. It seems like something changed (and of courses a lot of things changed!) but we can no longer depend on getting our JSString back so we can look it up in our list.
20:44
