<gsvelto>
FYI regarding my issue with a JIT test I may have found a fix that solves a real problem though a very sad one. The JS shell uses fork() to spawn new processes however that's not safe to do within our code on macOS. The reason is that fork() allocates memory and we're replacing the memory allocator. If the fork happens at the wrong time it can leave the memory allocator's structures - and non-fork-safe locks in particular - in an inconsistent state which leads to deadlocks or crashes.
